## 什么是「观察者模式」？

> 观察者观察被观察者，被观察者通知观察者

为了便于理解我们先使用“订阅通知”的(字面)概念来看「观察者模式」，接着我们翻译下上面「观察者模式」的概念：

- 两个对象
    + 被观察者 -> 主题
    + 观察者 -> 订阅者
- 两个动作
    + 订阅 -> 订阅者**订阅**主题
    + 通知 -> 主题发生变动**通知**订阅者

观察者模式的优势：

- 高内聚 -> 不同业务代码变动互不影响
- 可复用 -> 新的业务(就是新的订阅者)订阅不同接口(主题，就是这里的接口)
- 极易扩展 -> 新增接口(就是新增主题)；新增业务(就是新增订阅者)；
